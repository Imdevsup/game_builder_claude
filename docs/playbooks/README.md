# Playbooks

Named procedures for recurring or high-stakes situations. When a situation matches a playbook, follow it. Do not improvise around a known pattern.

- [new-feature.md](./new-feature.md), start of any feature work.
- [bug-fix.md](./bug-fix.md), bug reported, scoped, fixed, retro.
- [perf-issue.md](./perf-issue.md), slow code, memory leak, frame drops.
- [llm-cost-overrun.md](./llm-cost-overrun.md), daily budget breached.
- [exploit-found.md](./exploit-found.md), cheat or dupe discovered.
- [migration.md](./migration.md), schema or data migration.
- [deploy.md](./deploy.md), shipping a build to production.
- [incident.md](./incident.md), a server is on fire.
- [bible-edit.md](./bible-edit.md), when changing the bible.
