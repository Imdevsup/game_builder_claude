# Prompt style

The voice and structure of prompts shipped to the model.

## Per-prompt template

Every prompt template lives in `packages/shared/src/llm/prompts/<name>.ts` and exports:

```ts
{
  key: PromptKey,
  system: string,             // role + style + constraints
  userTemplate: (params) => string,
  responseSchema: ZodSchema,  // strict
  cacheTtlSeconds: number,
  fallback: (params) => InferType<typeof responseSchema>,
  costBudget: { tokensPerCall: number, callsPerDay: number },
}
```

## Style rules per voice

### System voice

- Sparse. 1-3 sentences.
- Present tense.
- No second-person address. The System never says "you." It says "the Awakened" or refers by class.
- No exclamation. No emoji.
- Words like "registered," "observed," "permitted," "denied."
- Example: `Registered. The Awakened named Mira holds Iron x12. The Market notes shifting weight.`

### NPC voice

- Three personality adjectives provided as parameters.
- One verbal tic provided per NPC. The tic appears once per response on average.
- Length: 1-4 sentences for normal turn. Up to 8 sentences if the player asks a backstory question.
- Always reference what the NPC actually has for sale or buys, when relevant.
- Never break character with meta-references.

### Price narration

- Single sentence.
- Names the item, the direction, and one cause.
- Never gives the actual number. The UI shows the number.

### Tailored class

- Name: 1-3 words, evocative, never duplicates an existing class.
- Flavor: 2-3 sentences. Mentions one piece of player history that shaped it.
- Mechanics block is generated by the deterministic class-architect engine, not the LLM. The LLM only writes the flavor.

## What prompts must never do

- Reproduce copyrighted text.
- Refer to the player's real-life identity.
- Generate slurs or graphic content.
- Reveal hidden state (boss identity, market thresholds, RNG seeds).
- Promise mechanics the engine does not support.
